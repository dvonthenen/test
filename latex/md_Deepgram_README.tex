\hypertarget{md_Deepgram_README_autotoc_md62}{}\doxysection{Logging}\label{md_Deepgram_README_autotoc_md62}
The \mbox{\hyperlink{classLibrary}{Library}} uses Microsoft.\+Extensions.\+Logging to preform all of its logging tasks. To configure logging for your app simply create a new {\ttfamily ILogger\+Factory} and call the {\ttfamily Log\+Provider.\+Set\+Log\+Factory()} method to tell the Deepgram library how to log. For example, to log to the console with Serilog, you\textquotesingle{}d need to install the Serilog package with {\ttfamily dotnet add package Serilog} and then do the following\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{using Microsoft.Extensions.Logging;}
\DoxyCodeLine{using Microsoft.Extensions.Configuration;}
\DoxyCodeLine{using Deepgram.Logger;}
\DoxyCodeLine{using Serilog;}
\DoxyCodeLine{}
\DoxyCodeLine{var log = new LoggerConfiguration()}
\DoxyCodeLine{    .MinimumLevel.Debug()}
\DoxyCodeLine{    .WriteTo.Console(outputTemplate: "{}\{Timestamp:HH:mm\} [\{Level\}]: \{Message\}\(\backslash\)n"{})}
\DoxyCodeLine{    .CreateLogger();}
\DoxyCodeLine{var factory = new LoggerFactory();}
\DoxyCodeLine{factory.AddSerilog(log);}
\DoxyCodeLine{LogProvider.SetLogFactory(factory);}

\end{DoxyCode}
 The sdk will generate loggers with the caterory\+Name of the client being used for example to get the logger for the \mbox{\hyperlink{classManageClient}{Manage\+Client}} you would call


\begin{DoxyCode}{0}
\DoxyCodeLine{LogProvider.GetLogger(nameof(ManageClient));}
